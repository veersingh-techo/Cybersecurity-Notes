# Introduction to Cybersecurity

## 1. Core Security Principles (CIA Triad)

### Confidentiality
Ensure data is accessible only to authorized users.

**Controls**
- Encryption
- Access Control
- Authentication
- MFA

### Integrity
Ensure data is accurate and unmodified.

**Controls**
- Hashing
- Checksums
- Digital Signatures

### Availability
Ensure systems and data remain accessible.

**Controls**
- Backups
- Redundancy
- Disaster Recovery
- DDoS Protection

---

## 2. Threat Landscape

### Malware

| Type | Purpose |
|--------|---------|
| Virus | Requires host file to spread |
| Worm | Self-replicates over networks |
| Trojan | Disguised as legitimate software |
| Spyware | Steals user information |
| Ransomware | Encrypts data for ransom |

### Social Engineering

Attacking humans instead of systems.

#### Common Techniques

- Phishing
- Spear Phishing
- Vishing
- Smishing
- Baiting
- Tailgating

### Denial of Service (DoS)

Goal:
- Exhaust system resources
- Make services unavailable

#### Types

- DoS
- DDoS

---

## 3. Types of Cybercriminals

### Script Kiddies
Use existing tools with limited knowledge.

### Black Hat Hackers
Unauthorized attackers.

### Organized Crime Groups
Financially motivated attackers.

### Nation-State Actors
Government-sponsored attackers.

### Insider Threats
Current or former employees abusing access.

### Hacktivists
Attack for political or social causes.

## 4. Data Protection

### Data States

#### Data at Rest
Stored data.

Examples:
- Database
- Hard Drive

#### Data in Transit
Moving across networks.

Examples:
- HTTPS Traffic
- Emails

#### Data in Use
Actively processed data.

Examples:
- Running Applications
- Open Documents

### Protection Methods

- Encryption
- Access Control
- Backups
- Data Classification

---

## 5. Authentication & Authorization

### Authentication

Verifying identity.

Examples:
- Password
- Fingerprint
- OTP

### Authorization

Determines permissions after authentication.

Examples:
- User
- Admin
- Root

### AAA Model

#### Authentication
Who are you?

#### Authorization
What can you access?

#### Accounting
What actions did you perform?

Filters incoming and outgoing traffic.

### VPN

## 9. Risk Management

### Risk Formula

Risk = Threat × Vulnerability × Impact

### Key Terms

#### Asset
Anything valuable.

Examples:
- Data
- Systems
- Reputation

#### Threat
Potential danger.

#### Vulnerability
Weakness that can be exploited.

#### Exploit
Method used to abuse a vulnerability.

---

## 10. Cybersecurity Best Practices

### Password Security

- Use Password Managers
- Unique Passwords
- MFA Everywhere

### Software Security

- Regular Updates
- Patch Management
- Remove Unused Software

### Email Security

- Verify Sender
- Avoid Unknown Links
- Check Attachments

### Device Security

- Screen Lock
- Full Disk Encryption
- Backups
