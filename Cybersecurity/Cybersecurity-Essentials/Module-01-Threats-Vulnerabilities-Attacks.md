# Module 1: Cybersecurity Threats, Vulnerabilities, and Attacks

## Threat Domains

### Threat Categories
- **Software Attacks** – Exploit software vulnerabilities to compromise systems.
- **Human Error** – Mistakes made by users that create security risks.
- **Sabotage** – Intentional damage to systems or data.
- **Theft** – Unauthorized stealing of information or devices.
- **Hardware Failure** – Physical component failures affecting security.
- **Utility Interruption** – Power or communication outages impacting systems.
- **Natural Disasters** – Floods, earthquakes, fires, and other natural events.

### Threat Sources
- **Internal Threats** – Employees, contractors, or insiders with authorized access.
- **External Threats** – Attackers attempting unauthorized access from outside.
- **User Domain** – Risks caused by insecure user behavior and weak security practices.

### Important Threats
- **APT (Advanced Persistent Threat)** – Long-term targeted attack by skilled attackers.
- **Backdoor** – Hidden method used to bypass authentication.
- **Rootkit** – Malware that hides itself and gains privileged access.
- **Dark Web** – Encrypted part of the internet not indexed by search engines.
- **IOC (Indicator of Compromise)** – Evidence that indicates a security breach.
- **STIX & TAXII** – Standards used for sharing cyber threat intelligence.

# Deception (Social Engineering)

Social engineering manipulates people into revealing confidential information or performing unsafe actions.

## Common Techniques
- **Phishing** – Fake emails or websites used to steal information.
- **Pretexting** – Creating a fake identity to obtain sensitive data.
- **Quid Pro Quo** – Offering something in exchange for confidential information.
- **Identity Theft** – Using another person's identity for fraud.
- **Shoulder Surfing** – Observing someone to steal sensitive information.
- **Dumpster Diving** – Recovering confidential data from discarded materials.
- **Piggybacking / Tailgating** – Entering restricted areas by following authorized users.
- **Watering Hole Attack** – Compromising websites frequently visited by victims.
- **Typosquatting** – Creating fake websites using misspelled domain names.

### Prevention
- Verify identities before sharing information.
- Follow organizational security policies.
- Conduct regular security awareness training.

# Cyber Attacks

## Malware
- **Virus** – Attaches to files and spreads when executed.
- **Worm** – Self-replicates across networks without user interaction.
- **Trojan Horse** – Appears legitimate while performing malicious actions.
- **Ransomware** – Encrypts data and demands payment.
- **Logic Bomb** – Malicious code triggered by a specific condition.

## Network Attacks
- **DoS (Denial of Service)** – Overloads a system to make it unavailable.
- **DDoS (Distributed DoS)** – Attack launched from multiple compromised devices.
- **DNS Spoofing** – Redirects users to fake websites.
- **DNS Hijacking** – Alters DNS settings to redirect traffic.

## Layer 2 Attacks
- **ARP Spoofing**
- **IP Spoofing**
- **MAC Flooding**
- **Man-in-the-Middle (MITM)**

## Other Attacks
- **Zero-Day Attack** – Exploits vulnerabilities before a fix is available.
- **Keylogger** – Records keystrokes to steal sensitive information.

### Defense
- Use firewalls.
- Apply software updates and security patches.
- Monitor network traffic.
- Block unnecessary external traffic.

# Wireless & Mobile Device Attacks

## Common Attacks
- **Grayware** – Unwanted software that negatively affects devices.
- **SMiShing** – Phishing through SMS messages.
- **Rogue Access Point** – Unauthorized wireless access point.
- **Evil Twin** – Fake Wi-Fi hotspot impersonating a legitimate network.
- **Radio Jamming** – Disrupts wireless communication signals.
- **Bluejacking** – Sending unwanted Bluetooth messages.
- **Bluesnarfing** – Stealing data through Bluetooth vulnerabilities.

## Wireless Security
- **WEP** – Older wireless security protocol (insecure).
- **WPA** – Improved wireless security protocol.
- **WPA2** – Stronger and widely used wireless encryption standard.

### Best Practices
- Change default passwords.
- Detect rogue access points.
- Use VPN on public Wi-Fi.
- Restrict guest network access.

# Application & Other Attacks

## Web Application Attacks
- **XSS (Cross-Site Scripting)** – Injects malicious JavaScript into web pages.
- **SQL Injection** – Executes unauthorized SQL queries.
- **XML Injection** – Manipulates XML input.
- **LDAP Injection** – Exploits LDAP queries.
- **Buffer Overflow** – Writes data beyond memory limits.
- **Remote Code Execution (RCE)** – Executes attacker-controlled code remotely.
- **CSRF** – Tricks authenticated users into performing unwanted actions.
- **Directory Traversal** – Accesses restricted files using manipulated paths.
- **API Attacks** – Exploits insecure API implementations.

## Email Attacks
- **Phishing**
- **Spear Phishing**
- **Whaling**
- **Vishing**
- **Pharming**

## Other Threats
- **Supply Chain Attack**
- **Cloud-Based Attack**
- **Adversarial AI Attack**

### Best Practices
- Validate all user input.
- Keep software updated.
- Scan email attachments before opening.
- Use antivirus and endpoint protection.
- Never trust unknown links or downloads.

---

# Key Takeaways

- Cybersecurity threats can originate from both internal and external sources.
- Human error remains one of the biggest security risks.
- Social engineering targets people rather than technology.
- Malware, network attacks, and web attacks are common attack vectors.
- Strong authentication, regular updates, user awareness, and layered security significantly reduce cyber risks.
